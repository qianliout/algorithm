# Dpds 项目需求 

第一部分：需求详细梳理

一、业务背景

┌─────────────────────────────────────────────────────────────────┐
│                        扫描引擎调度系统                          │
├─────────────────────────────────────────────────────────────────┤
│  场景：成千上万个待扫描任务持续流入                              │
│  特点：                                                          │
│    • 任务有不同优先级（P0-P4，P0 最高）                          │
│    • 任务有预估扫描时长                                          │
│    • 任务之间存在依赖关系（DAG）                                 │
│    • 依赖关系可能动态变动（添加/移除）                           │
│    • 任务持续到达，非一次性批处理                                │
│  目标：最大化高优先级任务的吞吐量                                │
└─────────────────────────────────────────────────────────────────┘

二、功能需求
编号   需求项   详细描述   优先级
F1   任务提交   支持动态提交新任务，包含优先级、预估时长、依赖列表   P0
F2   依赖管理   支持任务间依赖关系的动态添加和移除   P0
F3   优先级调度   严格按 P0>P1>P2>P3>P4 顺序调度   P0
F4   并发控制   限制同时运行的任务数量，避免资源过载   P0
F5   依赖感知   只有所有前置依赖完成的任务才能被调度   P0
F7   任务状态追踪   支持查询任务当前状态（PENDING/READY/RUNNING/COMPLETED）   P1
F8   动态依赖更新   任务运行中可添加/移除依赖关系   P1
F9   循环依赖检测   添加依赖时检测是否会形成环   P1
F10   优雅关闭   支持等待运行中任务完成后关闭调度器   P2

三、非功能需求
编号   需求项   指标要求
N1   性能   单次调度决策  80%
四、技术约束
语言：Go 1.21+
并发模型：goroutine + channel + sync.RWMutex
数据结构：
  可更新优先级堆（支持 O(log n) 优先级调整）
  依赖关系图（邻接表）
  任务状态机
线程安全：所有公共方法必须并发安全
依赖：仅使用标准库（无第三方依赖）
